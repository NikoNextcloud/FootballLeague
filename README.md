# А Група

Мобилно уеб приложение за мачове, резултати, класиране и отбори от българския футболен елит. Данните се зареждат от безплатния API на TheSportsDB (лига `4626`) и се пазят локално за по-бързо следващо отваряне.

## Отваряне на компютъра

Стартирайте `start-local.bat`. Не е нужна инсталация на програми или библиотеки (изисква наличен Python в Windows).

## Качване в GitHub Pages

1. Създайте ново публично хранилище в GitHub.
2. Качете съдържанието на тази папка в основната папка на хранилището.
3. Отворете **Settings → Pages**.
4. В **Build and deployment** изберете **Deploy from a branch**, `main` и `/ (root)`.
5. След запазване GitHub ще покаже адреса на приложението.

Приложението няма build стъпка и няма външни зависимости. Това го прави подходящо за директно публикуване в GitHub Pages.

Началната страница включва непрекъсната лента с емблемите. Всеки клуб има отделен профил със стадион, статистика, описание и мачове. Стрелките в класирането сравняват позицията с предходния изигран кръг.

## Автоматично обновяване и архив

GitHub Actions изпълнява `fetch-data.mjs` всеки ден. Ръчно може да се създаде архив за избран сезон с:

```text
node fetch-data.mjs 2025-2026
```

Полученият файл се записва в `data/2025-2026.json` и се използва от таб „Архив“.

## Известия

Бутонът с камбанка включва известия за начало на мач, докато приложението работи. Истински push известия при затворено приложение и моментални известия за гол изискват push сървър и live-score услуга. Безплатният TheSportsDB v1 API не предоставя live резултати в реално време.

## Българските отбори в Европа

Табът „Европа“ съдържа програмата на Левски в Шампионска лига, ЦСКА в Лига Европа и Лудогорец/ЦСКА 1948 в Лигата на конференциите. Екранът проверява за обновени резултати на всеки 60 секунди. Точните часове се показват след публикуването им от свързания източник.

## AiScore live панел

Когато започне мач от българската Първа лига, началният банер автоматично се заменя с вградената AiScore страница за Bulgarian First League. Панелът остава активен от пет минути преди началото до приблизително 165 минути след началния час. За проверка без изчакване може локално да се отвори `?live-preview=1`.
